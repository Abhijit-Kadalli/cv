Designing Real‑Time EEG Pipelines with LSL

October 2025

Building real‑time EEG applications demands tight synchronization, robust artifact rejection, and efficient feature extraction. This note outlines a practical pipeline using Lab Streaming Layer (LSL) to synchronize EEG with events and sensors.

Key steps

  • Inlet creation and stream discovery for EEG and event markers.
  • Timestamp correction and clock synchronization.
  • Band‑limited filtering and artifact rejection (blink/muscle).
  • Low‑latency features: Hjorth parameters, sample entropy, band power.
  • Sliding‑window batching and model inference.

Future posts will detail windowing strategies, jitter handling, and deployment.

System flow

sequenceDiagram
    participant User
    participant Frontend (React)
    participant Electron
    participant Backend (Django)
    participant LSL Outlet
    participant EEG Device
    participant LabRecorder (or other LSL client)

    User->>Electron: Launches App
    Electron->>Backend (Django): Spawns backend process
    Electron->>Frontend (React): Loads UI

    User->>Frontend (React): Clicks "Connect" with settings
    Frontend (React)->>Backend (Django): POST /api/connect/
    Backend (Django)->>EEG Device: Establishes connection
    EEG Device-->>Backend (Django): Connection OK
    Backend (Django)-->>Frontend (React): HTTP 200 OK { "success": true }
    Frontend (React)-->>User: UI navigates to Experiment page

    User->>Frontend (React): Clicks "Start Experiment"
    Frontend (React)->>Backend (Django): POST /api/sandbox/start_experiment/
    Backend (Django)->>LSL Outlet: Creates LSL Stream (e.g., EEG data, markers)
    LabRecorder (or other LSL client)->>LSL Outlet: Discovers and subscribes to stream
    Frontend (React)->>Backend (Django): Establishes WebSocket connection to /ws/eeg/
    
    loop Real-time Data Streaming
        EEG Device->>Backend (Django): Streams raw EEG data
        Backend (Django)->>LSL Outlet: Pushes parsed data to LSL stream with the assigned timestamp when recieving the raw eeg
        Backend (Django)->>Backend (Django): Processes data (filters, etc.)
        Backend (Django)->>Frontend (React): Pushes processed data via WebSocket
        LSL Outlet-->>LabRecorder (or other LSL client): Forwards data
        Frontend (React)->>User: Updates EEG plots in real-time
    end

    User->>Frontend (React): Clicks "Stop Experiment"
    Frontend (React)->>Backend (Django): POST /api/sandbox/stop_experiment/
    Backend (Django)->>LSL Outlet: Stops LSL stream
    Backend (Django)->>EEG Device: Halts data stream
    Backend (Django)-->>Frontend (React): HTTP 200 OK { "message": "..." }
